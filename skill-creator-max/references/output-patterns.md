# 出力フォーマットパターン

スキルの結果レポートで使える出力パターンのカタログ。

## 1. ステータスレポートパターン

最もシンプルなパターン。各項目の合否をアイコンで表示。

### 実例: deploy-test.md

```
## デプロイテスト結果
- URL: https://www.cmd-gen.com
- デスクトップ:     ✅ / ⚠️ <問題があれば記載>
- モバイル:         ✅ / ⚠️ <問題があれば記載>
- 機能テスト:       ✅ / ⚠️ <問題があれば記載>
- パフォーマンス:   ✅ / ⚠️ <指標と値を記載>
```

### アイコン規約

| アイコン | 意味 | 使用条件 |
|---------|------|----------|
| ✅ | 合格 | 基準値を満たしている |
| ⚠️ | 警告 | 基準値に近い、または軽微な問題 |
| ❌ | 不合格 | 基準値を超えている、重大な問題 |

## 2. メトリクスレポートパターン

定量値を一覧表示し、各値に対して判定を行う。

### 実例: perf-test.md

```
## パフォーマンステスト結果
- URL: <テスト対象URL>
- TTFB:               XXms ✅/⚠️
- FCP:                XXms ✅/⚠️
- DOMContentLoaded:   XXms ✅/⚠️
- Load Complete:      XXms ✅/⚠️
- 転送サイズ:          XX KB ✅/⚠️
- JS サイズ:           XX KB ✅/⚠️
- CLS:                X.XX ✅/⚠️
- インタラクション応答: ✅/⚠️
```

### 設計ポイント

- 値と判定アイコンをセットで表示
- 単位を明記（ms, KB, etc.）
- ⚠️ がある場合のみ詳細を追記

## 3. デプロイサマリーパターン

デプロイ結果を構造化して報告する。

### 実例: deploy.md

```
✅ デプロイ完了
- Commit: <コミットハッシュ> <コミットメッセージ>
- CI:     成功（所要時間）
- URL:    https://...vercel.app
```

### 設計ポイント

- 冒頭に結果アイコン + 1行サマリー
- 詳細情報はインデントして箇条書き
- 次のアクション（deploy-test の実行など）を提案

## 4. 比較テーブルパターン

複数の条件・環境での結果を比較表示する。

### 実例: ui-test.md

```
| 項目 | デスクトップ | モバイル |
|------|------------|---------|
| PresetGrid 列数 | 4列 ✅ | 2列 ✅ |
| ActionButtons 位置 | CommandOutput 内 ✅ | 固定バー ✅ |
| 固定バー表示 | なし ✅ | あり ✅ |
| 横スクロール | なし ✅ | なし ✅ |
```

### 設計ポイント

- 期待値と実際の値を並べる
- 各セルに判定アイコンを付与
- 不合格のセルを目立たせる

## 5. エラーレポートパターン

失敗時に原因と対処を構造化して報告する。

### テンプレート

```
❌ <操作> に失敗しました

### 原因
<エラーメッセージまたは状態の説明>

### 対処
1. <具体的な修正手順>
2. <再実行の方法>
```

### 実例

```
❌ CI type check に失敗しました

### 原因
src/lib/ffmpeg/builder.ts:42 - error TS2345:
  Argument of type 'string' is not assignable to parameter of type 'number'

### 対処
1. `npm run check` でエラー詳細を確認
2. 該当ファイルの型定義を修正
3. 再度 `git push origin main` して CI を再実行
```

## 6. プログレスレポートパターン

長い処理の途中経過を段階的に報告する。

### テンプレート

```
## 進捗状況

✅ Step 1: <完了したステップ>
✅ Step 2: <完了したステップ>
🔄 Step 3: <実行中のステップ>（XX%）
⏳ Step 4: <未実行のステップ>
```

### アイコン規約

| アイコン | 意味 |
|---------|------|
| ✅ | 完了 |
| 🔄 | 実行中 |
| ⏳ | 待機中 |
| ❌ | 失敗 |
| ⏭️ | スキップ |

## 7. サマリー + 詳細パターン

冒頭にサマリー、続けて詳細を表示する。

### テンプレート

```
## テスト結果サマリー

**6/7 合格** (1 件警告)

| 項目 | 結果 |
|------|------|
| 型チェック | ✅ |
| ユニットテスト | ✅ |
| CSS lint | ✅ |
| i18n check | ✅ |
| UI デスクトップ | ✅ |
| UI モバイル | ⚠️ 横スクロール検出 |
| パフォーマンス | ✅ |

### ⚠️ 警告詳細

**UI モバイル**: 375px 幅でプリセットカードの右端が 2px はみ出し
→ `PresetGrid` の padding を調整する
```

## フォーマットの選択ガイド

| 状況 | 推奨パターン |
|------|------------|
| 単純な合否報告 | ステータスレポート |
| 数値指標の報告 | メトリクスレポート |
| デプロイ結果 | デプロイサマリー |
| マルチ環境比較 | 比較テーブル |
| 失敗の詳細報告 | エラーレポート |
| 長時間処理 | プログレスレポート |
| 複合テスト結果 | サマリー + 詳細 |
