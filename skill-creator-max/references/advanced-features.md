# 高度な機能パターン

基本的なスキル構造をマスターした後に取り入れられる、高度な機能パターン集。

## 1. 条件分岐パターン

実行時の状態に応じて手順を分岐させる。

### 実例: deploy.md の CI 結果分岐

```markdown
### 2. CI 完了待機

- `conclusion: success` → 次へ
- `conclusion: failure` → エラーを確認し、修正して再実行
```

### 応用: 環境検出による分岐

```markdown
## 手順

1. 実行環境を確認する
   - ローカル preview (`localhost:4173`): ビルド済みであること
   - 本番 (`www.cmd-gen.com`): デプロイ済みであること
2. 環境に応じてベース URL を設定する
```

## 2. マルチステージパターン

複数のフェーズを順番に実行し、各フェーズの結果を次に引き継ぐ。

### 実例: deploy.md の 4 ステージ

```
前提チェック → push → CI 待機 → URL 取得
```

### 設計のポイント

- 各ステージの成功条件を明記
- 失敗時のリカバリポイントを定義
- ステージ間のデータ受け渡し方法を明示

## 3. パラメータ化パターン

引数で動作をカスタマイズ可能にする。

### 実例: perf-test.md の URL パラメータ

```markdown
## 対象 URL

- **本番**: `https://www.cmd-gen.com`
- **ローカル**: `http://localhost:4173`

引数で URL が渡された場合はそちらを使用する。
```

### 設計のポイント

- デフォルト値を明記
- 引数の形式・制約を説明
- 引数なしでも動作する設計にする

## 4. クロスリファレンスパターン

他のスキルを内部で参照・呼び出す。

### 実例: ui-test.md → perf-test.md

```markdown
### 4. パフォーマンステスト

`/perf-test` スキルの手順に従い、ローカル preview でパフォーマンス計測を実行する。
```

### 実例: deploy-test.md → perf-test.md

```markdown
### 4. パフォーマンステスト

`/perf-test` スキルの手順に従い、本番 URL でパフォーマンス計測を実行する。
```

### 設計のポイント

- 参照先スキルが存在することを前提にしない（フォールバック）
- 参照先スキルの変更が影響しないよう、最小限の依存にする
- 循環参照を避ける

## 5. チェックリスト比較パターン

複数の条件をテーブルで整理し、一覧性を高める。

### 実例: ui-test.md のレスポンシブ比較

```markdown
| 項目 | デスクトップ | モバイル |
|------|------------|---------|
| PresetGrid 列数 | 4列 | 2列 |
| ActionButtons 位置 | CommandOutput 内 | 固定バー |
| 固定バー表示 | なし | あり |
```

## 6. 数値ベンチマークパターン

定量的な合否基準を定義する。

### 実例: perf-test.md の合格基準

```markdown
| 指標 | 目標値 | 備考 |
|------|--------|------|
| TTFB | < 200ms | Vercel Edge |
| FCP | < 1000ms | 初回描画 |
| CLS | < 0.1 | レイアウトシフト |
```

### 設計のポイント

- 目標値の根拠を備考に記載
- 計測方法を明確に定義（コード例付き）
- 結果のレポート形式に ✅/⚠️ を使用

## 7. スクリプト実行パターン

コードブロック内にスクリプトを定義し、実行手順を自動化する。

### 実例: perf-test.md の JavaScript 計測コード

計測に必要なコードを完全な形でコードブロックに含め、
Claude がそのまま実行できるようにする。

### 設計のポイント

- コードは完全でコピーペースト可能であること
- 言語タグ（```bash, ```javascript）を必ず付ける
- 実行結果の解釈方法も併記する

## 8. 段階的詳細化パターン

最初はハイレベルな概要を示し、サブセクションで詳細を掘り下げる。

### 実例: ui-test.md の構造

```
## テスト手順
  ### 1. デスクトップテスト (1280x900)
  ### 2. モバイルテスト (375x812)
  ### 3. 機能テスト
  ### 4. パフォーマンステスト
```

### 設計のポイント

- H2 で大分類、H3 で詳細ステップ
- H3 には具体的な数値・条件を含める
- 各 H3 が独立して実行可能な粒度にする
